# PDF Report Generation Module — Implementation Plan

## Background

DefenSYS manages capstone and PIT defense workflows including academic periods, student teams, defense scheduling, grading (panel + adviser + peer), rubrics, repository deliverables, and audit compliance. The system already has **ReportLab ≥4.0** installed (used by [pdf_generator.py](file:///c:/Users/Admin/Desktop/DefenSYS/backend/modules/repository/deliverables/pdf_generator.py) for weekly progress report compilation), but there is **no system-wide report export feature** yet.

The existing `pdf_generator.py` establishes a good pattern (maroon headers `#7F1D1D`, `Helvetica-Bold`, `letter` page size, cover pages, `TableStyle` with grid lines) that we'll extend to all new reports for brand consistency.

---

## Open Questions

1. **Report Selection**: The 6 report types below are based on all available data models. If you want to add/remove any, or prioritize certain ones, please flag them.
2. **Institution Logo**: Should the PDFs include a university/department logo on cover pages? If so, where is the logo file? (I can use `assets/logo.png` from the Flutter project if appropriate.)
3. **Access Control**: Currently planned as admin-only exports. Should faculty (PIT leads, advisers) also be able to export reports scoped to their teams/cohort?
4. **Report naming convention** — Should exported PDFs use filenames like `DefenSYS_Grade_Report_TeamName_2026-05-29.pdf` or something simpler?
5. **Watermark / Confidential stamp** — Should grade reports include a "CONFIDENTIAL" watermark?
6. **Digital signature** — Do you need a "Generated by DefenSYS" footer with timestamp on every page, or just the cover page?

## Proposed Report Types

| # | Report Name | Data Source | Scope | Description |
|---|-------------|-------------|-------|-------------|
| 1 | **Team Grade Report Card** | `TeamGrade`, `GradeBreakdown`, `PanelistGradeSubmission`, `StudentPeerGrade` | Per-team | Full breakdown of panel/adviser/peer scores per defense stage, criterion-level detail, final weighted grade |
| 2 | **Semester Grade Summary** | `TeamGrade`, `StudentTeam`, `Semester` | Per-semester | All teams' grades in a single table — stage, panel score, adviser score, peer score, final grade, result (passed/failed) |
| 3 | **Defense Schedule Summary** | `DefenseSchedule`, `SchedulePanelist`, `DefenseStage` | Per-semester or date range | All scheduled defenses with team, stage, date, time, room, panelists |
| 4 | **Team Roster Report** | `StudentTeam`, `TeamMembership`, `User` | All teams or filtered | Team name, project title, level, year level, leader, adviser, members, status |
| 5 | **User Directory** | `User`, role flags | All users or filtered | ID, name, email, role, active status, faculty sub-roles |
| 6 | **Audit Trail Export** | `SystemAuditLog` | Date range + category | Action, actor, category, target, old/new values, timestamp |

## Proposed Changes

### Backend — New `reports` module

A new Django app `reports` under `backend/modules/reports/` that owns all PDF generation logic and API endpoints.

---

#### [NEW] `reports/__init__.py`
Empty init.

#### [NEW] `reports/apps.py`
Standard Django `AppConfig` with `name = 'reports'`.

#### [NEW] `reports/urls.py`
URL patterns mapping each report type to its view:
```
reports/team-grade/<int:team_id>/
reports/semester-grades/
reports/defense-schedules/
reports/team-roster/
reports/user-directory/
reports/audit-trail/
```

#### [NEW] `reports/views.py`
Six `APIView` subclasses, each:
- Accepts query parameters for filtering (semester ID, date range, role, etc.)
- Calls the appropriate service function to gather data
- Calls the appropriate PDF generator
- Returns `HttpResponse(content_type='application/pdf')` with `Content-Disposition: attachment`
- Permission: `IsAuthenticated, IsAdminRole` (initially)

#### [NEW] `reports/pdf_styles.py`
Shared PDF styling utilities extracted from patterns in the existing `pdf_generator.py`:
- `defensys_styles()` — returns a style dict with Title, Subtitle, Heading, Body, Footer
- `defensys_cover_page(story, title, subtitle, metadata_rows)` — reusable cover page builder
- `defensys_table_style()` — standard maroon-header table style
- `defensys_page_footer(canvas, doc)` — "Generated by DefenSYS on {date}" footer on every page
- Brand constants: maroon `#7F1D1D`, gold `#D4A843`, page size `letter`

#### [NEW] `reports/generators/team_grade_report.py`
**Team Grade Report Card** generator:
- Cover page: Team name, project title, year level, adviser, semester, generation date
- Per defense stage section:
  - Summary: panel score, adviser score, peer score, weights, final grade
  - Panel breakdown table: criterion name, score, max score, normalized %, per-panelist scores
  - Adviser score (if capstone)
  - Peer evaluation summary per member
- Final result page with overall status (Passed/Failed/Pending)

#### [NEW] `reports/generators/semester_grades_report.py`
**Semester Grade Summary** generator:
- Cover page: Semester label, school year, generation date
- Summary statistics: total teams, graded count, published count, average grade
- Main table: Team Name | Project Title | Stage | Panel | Adviser | Peer | Final Grade | Result
- Sorted by team level then name

#### [NEW] `reports/generators/defense_schedule_report.py`
**Defense Schedule Summary** generator:
- Cover page with semester/date-range filter info
- Grouped by date:
  - Date header
  - Table: Time | Room | Team | Stage/Event | Panelists | Status

#### [NEW] `reports/generators/team_roster_report.py`
**Team Roster Report** generator:
- Cover page with filter info (level, year level, status)
- Table: Team Name | Project Title | Level | Year Level | Leader | Adviser | Members | Status
- Summary row: totals per status

#### [NEW] `reports/generators/user_directory_report.py`
**User Directory** generator:
- Cover page
- Table: ID | Name | Email | Role | Sub-Roles | Status
- Grouped or filtered by role

#### [NEW] `reports/generators/audit_trail_report.py`
**Audit Trail Export** generator:
- Cover page with date range and category filter
- Table: Timestamp | Actor | Action | Category | Target | Details
- Page-broken by date

#### [NEW] `reports/generators/__init__.py`
Empty init.

## Phased Delivery

### Phase 1 — Foundation (Backend)
**Effort: ~2-3 hours**

- Create the `reports` Django app scaffold (`__init__`, `apps`, `urls`)
- Build `pdf_styles.py` shared utilities
- Wire into root `urls.py` and `settings.py`
- Implement **Team Grade Report Card** generator + view (most complex, proves the pattern)
- Test with a real team via `curl` or browser

### Phase 2 — Remaining Generators (Backend)
**Effort: ~2-3 hours**

- Semester Grade Summary
- Defense Schedule Summary
- Team Roster Report
- User Directory
- Audit Trail Export

### Phase 3 — Reports Screen (Frontend)
**Effort: ~2-3 hours**

- Add sidebar entry + route
- Build the Report Center UI with report-type cards and filter panels
- Implement `reports_provider.dart` with download logic
- Wire up PDF generation and browser download

### Phase 4 — Contextual Export Buttons (Frontend)
**Effort: ~1-2 hours**

- Add "Export PDF" buttons to Grade Center, Defense Scheduler, Student Teams, User Management, Audit Trail screens
- Each button calls the same API, shows loading spinner, downloads the file

### Phase 5 — Polish & Edge Cases
**Effort: ~1 hour**

- Empty state handling (no data → informative message page instead of empty PDF)
- Large dataset pagination (>500 rows → multi-page tables with repeated headers)
- Error toasts for API failures
- File naming convention with timestamps
